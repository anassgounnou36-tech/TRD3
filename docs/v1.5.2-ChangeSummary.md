# v1.5.2 Change Summary

- Fixed execution preflight false negatives where `OrderCheck()` returned `true` but `retcode=0` (tester behavior): this is now treated as acceptable preflight success.
- Added dedicated request price sanitization so execution uses broker-aware, symbol-digit-normalized entry/SL/TP before preflight and send.
- Replaced early raw stop-side rejection with sanitize-first stop normalization (BUY stop below entry, SELL stop above entry, with minimum stop-distance enforcement).
- Improved execution diagnostics to show raw signal prices, snapped entry, final normalized prices, stop-distance constraints, and detailed OrderCheck results.
